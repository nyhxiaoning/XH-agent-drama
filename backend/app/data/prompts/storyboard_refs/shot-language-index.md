## 唯一事实源原则（先读）

- 【整合决策矩阵】是 **景别 / 焦段 / 运镜 / 节奏** 四维的**唯一事实源**。
- 快速检索卡、`shot-language-detail.md` 的各模块明细、焦段速查表，均为**派生视图**；与矩阵冲突时**以矩阵为准**。
- 新增 / 修改条目时：**先改整合矩阵，再同步派生视图**（快速检索卡关键词、明细模块对应条目）。
- 一致性自检（引用条目时默想）：该条目码是否在 ①快速检索卡 ②整合矩阵 ③明细模块 三处都存在且四维一致？缺一即为漂移，需补齐。

## 查询流程（每帧执行 · 两轴模型）

```
情绪基底轴（A–G，必选） + 类型叠加轴（H，仅动作场景）

① 识别【情绪基底】→ 查【快速检索卡·情绪】定位 A–G 条目码
   → 读【整合决策矩阵】对应行，取 景别/焦段/运镜/节奏 作为四维起点
② 判断是否【动作类型场景】（武打/打斗/枪战/飙车）
   → 是：查【快速检索卡·类型】定位 H 码，读【类型叠加层速查】取动作骨架
   → 否：跳过，仅用情绪基底
③ 叠加合成（仅当存在 H）：
   类型层的 景别/机位/运镜/构图 = 动作骨架；
   情绪基底的 光影/表演/节奏 = 情绪着色；
   冲突时 → 动作可读性 > 情绪渲染。
④ 需 机位/构图/光影/禁忌 → 读 detail：模块一(情绪) / 类型叠加层明细(类型)
⑤ 验证：运镜意图(detail 模块二) / 景别节奏(detail 模块三) / 焦段心理(detail 模块四)
⑥ 局部冲突优先级 = 焦段 > 景别 > 运镜
```
